=======================================================
PART 2: ONE-TIME SETUP (do this once ever)
=======================================================

STEP 1 — Install Git
─────────────────────
Mac:
  Open Terminal → type: git --version
  If not installed, Mac will prompt you to install it. Click Install.

Windows:
  Go to: https://git-scm.com/download/win
  Download and install with all default options.

Linux:
  sudo apt install git

STEP 2 — Create a GitHub account
──────────────────────────────────
Go to: https://github.com
Click "Sign up" → create account with your email.
Verify your email when prompted.

STEP 3 — Configure Git with your identity
───────────────────────────────────────────
Open Terminal (Mac/Linux) or Git Bash (Windows) and run:

  git config --global user.name "Your Full Name"
  git config --global user.email "your@email.com"

(Use the same email as your GitHub account.)


=======================================================
PART 3: CREATE THE GITHUB REPOSITORY
=======================================================

STEP 4 — Create a new repo on GitHub
──────────────────────────────────────
1. Go to: https://github.com
2. Click the GREEN "New" button (top left) OR go to: https://github.com/new
3. Fill in:
     Repository name:  missed-connections-prediction
     Description:      AI-powered missed flight connection prediction & decision intelligence system for airline operations
     Visibility:       ✅ Public  (so recruiters can see it)
     Add a README:     ❌ NO  (we already have one)
     Add .gitignore:   ❌ NO  (we'll create one)
     License:          MIT (optional, recommended)

4. Click "Create repository"
5. GitHub shows you a page with your repo URL — copy it:
   Example: https://github.com/yourname/missed-connections-prediction.git
   KEEP THIS PAGE OPEN — you'll need the URL in a moment.

=======================================================
PART 5: PUSH TO GITHUB — TERMINAL COMMANDS
=======================================================

Open Terminal (Mac/Linux) or Git Bash (Windows).

STEP 8 — Navigate to your project folder
──────────────────────────────────────────
  cd ~/Desktop/missed-connections-prediction

(If your folder is elsewhere, adjust the path accordingly)

Confirm you're in the right place:
  ls
You should see: README.md  requirements.txt  docs/  data/  src/  tests/

STEP 9 — Initialize Git in this folder
─────────────────────────────────────────
  git init

You'll see: Initialized empty Git repository in .../missed-connections-prediction/.git/

STEP 10 — Connect to your GitHub repo
───────────────────────────────────────
  git remote add origin https://github.com/YOURUSERNAME/missed-connections-prediction.git

Replace YOURUSERNAME with your actual GitHub username.

STEP 11 — Add all files
─────────────────────────
  git add .

This stages ALL files for commit. The dot means "everything in this folder."

STEP 12 — Commit (take a snapshot)
────────────────────────────────────
  git commit -m "Initial commit: ConnectGuard - Missed Connection Prediction System

  - Full PRD and product thinking documentation
  - Synthetic data generator for FRA hub simulation
  - Feature engineering pipeline (27 features)
  - Hybrid rule + GBM risk scoring model
  - Decision engine with cost-benefit analysis
  - Streamlit operations dashboard
  - Unit and integration tests"

STEP 13 — Push to GitHub
─────────────────────────
  git branch -M main
  git push -u origin main

GitHub will ask for your username and password.
⚠️ NOTE: GitHub no longer accepts your account password here.
You need a Personal Access Token instead. See STEP 14.

STEP 14 — Generate a GitHub Personal Access Token (if needed)
───────────────────────────────────────────────────────────────
1. Go to: https://github.com/settings/tokens
2. Click "Generate new token" → "Generate new token (classic)"
3. Note/name: "ConnectGuard upload"
4. Expiration: 90 days
5. Scopes: check "repo" (full control of private repositories)
6. Click "Generate token"
7. COPY THE TOKEN NOW — GitHub won't show it again
8. When Terminal asks for password → paste the token (not your account password)

STEP 15 — Verify it worked
─────────────────────────────
Go to: https://github.com/YOURUSERNAME/missed-connections-prediction

You should see:
✅ Your README.md rendered beautifully as the project homepage
✅ All folders visible: docs/, data/, src/, tests/
✅ File count and last commit message

=======================================================
PART 10: TROUBLESHOOTING
=======================================================

Problem: "fatal: remote origin already exists"
Solution: git remote set-url origin https://github.com/YOURUSERNAME/missed-connections-prediction.git

Problem: "Permission denied" or authentication error
Solution: Use Personal Access Token as password (see STEP 14)

Problem: "src/dashboard/app.py not found" on Streamlit
Solution: Make sure the file path in Streamlit Cloud exactly matches your repo structure

Problem: Streamlit app crashes on launch
Solution: Make sure requirements.txt includes all dependencies, then redeploy

Problem: Large files rejected by GitHub (>100MB)
Solution: data/synthetic/ CSVs are in .gitignore so they won't be pushed
          The data is generated locally by running: python data/generate_synthetic_data.py
